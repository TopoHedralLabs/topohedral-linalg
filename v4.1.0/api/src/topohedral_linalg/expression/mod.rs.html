1//! Expression-template infrastructure for lazy, allocation-free arithmetic.
2//!
3//! Re-exports the two sub-modules that together implement lazy evaluation for matrix arithmetic:
4//! [`binary_expr`] for operations between two operands (addition, subtraction, element-wise
5//! multiplication/division) and [`unary_expr`] for single-operand transformations (negation).
6//! Results are not evaluated until the expression is converted into a concrete matrix type,
7//! letting the compiler fuse chains of operations into a single pass without intermediate heap
